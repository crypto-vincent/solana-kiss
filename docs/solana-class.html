Constructor

Create a Solana instance by passing either a network moniker, a URL, or a pre-built RpcHttp client.

new Solana(rpcHttpOrUrl, options?)
ParameterTypeDescription
rpcHttpOrUrlRpcHttp | URL | stringA moniker ("mainnet", "devnet", "testnet"), a URL, or an existing RpcHttp client.
options.idlLoaderIdlLoader?Custom IDL loader. Defaults to: native on-chain → Anchor on-chain → GitHub fallback.
options.idlOverridesMap<Pubkey, IdlProgram>?Hardcoded IDLs that bypass the loader entirely.
options.recentBlockHashCacheDurationMsnumber?How long to cache the recent block hash. Defaults to 15_000 ms.
 
import { Solana, rpcHttpFromUrl, rpcHttpWithTimeout } from "solana-kiss";

// 1. Moniker shorthand (easiest)
const solana = new Solana("mainnet");

// 2. Custom RPC URL
const solana2 = new Solana(new URL("https://my-rpc.example.com"));

// 3. Pre-built client — add a 5-second timeout
const client = rpcHttpWithTimeout(
  rpcHttpFromUrl(new URL("https://my-rpc.example.com")),
  5_000
);
const solana3 = new Solana(client);

// 4. With options
const solana4 = new Solana("devnet", {
  recentBlockHashCacheDurationMs: 30_000, // cache block hash for 30 s
});

IDL Management

solana-kiss resolves program IDLs automatically. You can also override them for local testing or unsupported programs.

Loading an IDL

solana.getOrLoadProgramIdl(programAddress, options?)

Returns { programIdl: IdlProgram }. The result is cached per address.

const { programIdl } = await solana.getOrLoadProgramIdl(
  "JUP6LkbZbjS1jKKwapdHNy74zcZ3tLUZoi5QNyVTaV4" as any // Jupiter v6
);

console.log(programIdl.metadata.name);     // "jupiter"
console.log([...programIdl.instructions.keys()]); // ["route", "sharedAccountsRoute", ...]

Registering an IDL Override

Useful when testing against a local validator or when the on-chain IDL is not available. 

import { Solana, idlProgramParse } from "solana-kiss";
import myIdlJson from "./my-program.json";

const solana = new Solana("devnet");

// Parse raw JSON into an IdlProgram and register it
const myIdl = idlProgramParse(myIdlJson);
solana.setProgramIdlOverride(
  "MyProgram1111111111111111111111111111111111" as any,
  myIdl
);

// Now all calls to the program will use myIdl instead of fetching on-chain
const { instructionRequest } = await solana.hydrateAndEncodeInstruction(
  "MyProgram1111111111111111111111111111111111" as any,
  "initialize",
  { instructionAddresses: { authority: payer.address }, instructionPayload: {} }
);

Getting a Single Instruction's IDL

solana.getOrLoadInstructionIdl(programAddress, instructionName)
const { instructionIdl } = await solana.getOrLoadInstructionIdl(
  "JUP6LkbZbjS1jKKwapdHNy74zcZ3tLUZoi5QNyVTaV4" as any,
  "route"
);

// Inspect accounts and arguments
instructionIdl.accounts.forEach(a =>
  console.log(a.name, "writable:", a.writable, "signer:", a.signer)
);

Account Decoding

getAndInferAndDecodeAccount fetches raw account bytes, identifies the owning program, loads its IDL, guesses the account type by matching the discriminator, and returns the fully decoded state — all in one call.

solana.getAndInferAndDecodeAccount(accountAddress)
Return fieldTypeDescription
programAddressPubkeyAddress of the owning program.
programIdlIdlProgramResolved IDL of the owning program.
accountAddressPubkeyThe queried address.
accountIdlIdlAccountThe matched account type from the IDL.
accountLamportsbigintCurrent lamport balance.
accountExecutablebooleanWhether the account holds executable code.
accountDataUint8ArrayRaw account bytes.
accountStateJsonValueDecoded account fields as a plain JSON object.
 
import { Solana } from "solana-kiss";

const solana = new Solana("mainnet");

// Decode a Serum / OpenBook market account
const result = await solana.getAndInferAndDecodeAccount(
  "9wFFyRfZBsuAha4YcuxcXLKwMxJR43S7fPfQLusDBzvT" as any
);

console.log("Program:", result.programAddress);
console.log("Account type:", result.accountIdl.name); // e.g. "market"
console.log("Balance:", result.accountLamports, "lamports");
console.log("State:", result.accountState);
// {
//   ownAddress: "9wFFy...",
//   vaultSignerNonce: 0n,
//   baseMint: "So11...",
//   quoteMint: "EPjF...",
//   baseVault: "...",
//   ...
// }
💡 Tip
This method does not throw when the account is uninitialized. It returns zeroed defaults (programAddress is the default pubkey, accountLamports is 0n).

Building Instructions

Encode a Full Instruction

hydrateAndEncodeInstruction resolves missing accounts automatically (PDAs, ATAs, system accounts) then encodes the full instruction ready for a transaction.

solana.hydrateAndEncodeInstruction(programAddress, instructionName, options)
import { Solana, signerGenerate } from "solana-kiss";

const solana = new Solana("devnet");
const payer = await signerGenerate();

// Build a "createAccount" instruction for the System Program
const { instructionRequest } = await solana.hydrateAndEncodeInstruction(
  "11111111111111111111111111111111" as any,  // System Program
  "createAccount",
  {
    instructionAddresses: {
      fundingAccount: payer.address,
      newAccount: newKeypair.address,
    },
    instructionPayload: {
      lamports: 1_000_000n,
      space: 165n,
      owner: "TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA" as any,
    },
  }
);

// instructionRequest is ready to pass to prepareAndExecuteTransaction

Decode a Raw Instruction

Given a raw InstructionRequest (e.g. from a parsed transaction), reverse-engineer it into a human-readable form.

solana.inferAndDecodeInstruction(instructionRequest)
const { programIdl, instructionIdl, instructionAddresses, instructionPayload }
  = await solana.inferAndDecodeInstruction(rawInstructionRequest);

console.log("Instruction:", instructionIdl.name);
console.log("Args:", instructionPayload);
console.log("Accounts:", instructionAddresses);

Hydrate Missing Accounts Only

Use hydrateInstructionAddresses to fill in only the account addresses without encoding the full instruction — useful for inspecting resolved PDAs before calling. 

const { instructionAddresses } = await solana.hydrateInstructionAddresses(
  myProgramAddress,
  "mintTo",
  {
    instructionAddresses: { authority: payer.address, mint: mintAddress },
    instructionPayload: { amount: 1000n },
  }
);

// All derivable accounts are now filled in
console.log("destination ATA:", instructionAddresses.destination);

Resolve a Single Account Address

solana.resolveInstructionAddress(programAddress, instructionName, accountName, options?)
// Find the exact PDA for a specific account in an instruction
const metadataPda = await solana.resolveInstructionAddress(
  metaplexProgramAddress,
  "createMetadataAccountV3",
  "metadata",
  { instructionAddresses: { mint: mintAddress } }
);

console.log("Metadata PDA:", metadataPda);

Executing Transactions

prepareAndExecuteTransaction handles everything: fetches a recent block hash (with caching), compiles a versioned transaction, collects all signatures, submits to the network, and waits for confirmation.

solana.prepareAndExecuteTransaction(payerSigner, instructionsRequests, options?)
Return fieldTypeDescription
transactionHandleTransactionHandleThe on-chain transaction signature.
transactionRequestTransactionRequestThe decompiled instruction inputs.
transactionPacketTransactionPacketThe raw signed wire bytes.
executionReportExecutionReportOn-chain metadata: slot, fees, logs, error.
executionFlowExecutionFlow | undefinedStructured call-stack trace parsed from logs.
 
import { Solana, signerGenerate } from "solana-kiss";

const solana = new Solana("devnet");
const payer = await signerGenerate();

// Assume we have two instruction requests already built
const ix1 = await solana.hydrateAndEncodeInstruction(...);
const ix2 = await solana.hydrateAndEncodeInstruction(...);

const {
  transactionHandle,
  executionReport,
  executionFlow,
} = await solana.prepareAndExecuteTransaction(payer, [ix1, ix2], {
  skipPreflight: false,
  skipExecutionFlowParsing: false, // parse program logs into call tree
});

console.log("Tx signature:", transactionHandle);
console.log("Slot:", executionReport.slot);
console.log("Fee:", executionReport.fee, "lamports");
console.log("Logs:", executionReport.logs);

if (executionFlow) {
  // Structured call-stack trace of all CPI calls
  console.log("Call tree:", executionFlow);
}

Transaction with Multiple Signers

const { transactionHandle } = await solana.prepareAndExecuteTransaction(
  payer,                             // fee payer + first signer
  [instructionRequest],
  {
    extraSigners: [coSigner, programSigner],  // additional keypairs
  }
);

Transaction with Address Lookup Tables

const { transactionHandle } = await solana.prepareAndExecuteTransaction(
  payer,
  [instructionRequest],
  {
    transactionLookupTables: [
      {
        tableAddress: "LookupTableAddress..." as any,
        lookupAddresses: [addr1, addr2, addr3],
      },
    ],
  }
);

Simulating Transactions

Simulate without broadcasting. Useful for gas estimation and dry-runs.

solana.prepareAndSimulateTransaction(payer, instructionsRequests, options?)
// Full simulation with signatures (most realistic)
const result = await solana.prepareAndSimulateTransaction(
  payer,  // Signer
  [instructionRequest]
);
console.log("Compute units:", result.unitsConsumed);
console.log("Logs:", result.logs);
if (result.error) console.error("Sim error:", result.error);

// Quick simulation without real signatures (no need for a Signer)
const quick = await solana.prepareAndSimulateTransaction(
  payer.address,  // Pubkey only
  [instructionRequest],
  { verifySignaturesAndBlockHash: false }
);

// Inspect post-simulation account state for specific addresses
const withAccounts = await solana.prepareAndSimulateTransaction(
  payer,
  [instructionRequest],
  {
    simulatedAccountsAddresses: new Set([mintAddress, ataAddress]),
  }
);
console.log("Post-sim mint state:", withAccounts.accounts);

PDA Derivation

Derive a Program Derived Address by name using the program's IDL PDA definition — no need to know the raw seeds.

solana.findPdaAddress(programAddress, pdaName, pdaInputs?)
// Derive the metadata PDA for a mint (Metaplex Token Metadata)
const metadataPda = await solana.findPdaAddress(
  "metaqbxxUerdq28cj1RbAWkYQm3ybzjb6a8bt518x1s" as any,
  "metadata",
  { mint: mintAddress }
);

console.log("Metadata PDA:", metadataPda);

// Derive a user's escrow account PDA (if the program IDL defines one)
const escrowPda = await solana.findPdaAddress(
  myProgramAddress,
  "userEscrow",
  { user: userAddress, market: marketAddress }
);

Querying Program Accounts

Fetch all on-chain accounts of a specific type owned by a program. Filters are derived automatically from the IDL (discriminator + data size).

solana.findProgramOwnedAccounts(programAddress, accountName)
// Find all "Position" accounts for a perpetuals program
const { accountsAddresses } = await solana.findProgramOwnedAccounts(
  perpProgramAddress,
  "position"
);

console.log(`Found ${accountsAddresses.size} positions`);

// Then decode each one
for (const address of accountsAddresses) {
  const { accountState } = await solana.getAndInferAndDecodeAccount(address);
  console.log(address, "→", accountState);
}

Direct RPC Access

Access the underlying RpcHttp client directly for any low-level RPC calls not covered by the high-level API. 

import { rpcHttpGetAccountLamports } from "solana-kiss";

const rpcHttp = solana.getRpcHttp();

// Use any low-level RPC helper
const { accountLamports } = await rpcHttpGetAccountLamports(
  rpcHttp,
  myAddress
);

// Or call the RPC directly
const slot = await rpcHttp("getSlot", [], {});
console.log("Current slot:", slot);

Recipes

01 Airdrop and check balance on devnet

Generate a keypair, request an airdrop, and confirm the lamport balance.

import { Solana, signerGenerate, rpcHttpGetAccountLamports } from "solana-kiss";

const solana = new Solana("devnet");
const keypair = await signerGenerate();
console.log("Address:", keypair.address);

// Request airdrop via raw RPC call
const rpc = solana.getRpcHttp();
await rpc("requestAirdrop", [keypair.address, 1_000_000_000], "skip-configuration-object");

// Wait a moment then check the balance
await new Promise(r => setTimeout(r, 3000));
const { accountLamports } = await rpcHttpGetAccountLamports(rpc, keypair.address);
console.log("Balance:", accountLamports, "lamports"); // 1000000000
02 Batch-decode multiple accounts

Decode many accounts in parallel using Promise.all.

const addresses = [addr1, addr2, addr3];

const results = await Promise.all(
  addresses.map(addr => solana.getAndInferAndDecodeAccount(addr))
);

results.forEach(({ accountIdl, accountState }, i) => {
  console.log(addresses[i], "type:", accountIdl.name, "state:", accountState);
});
03 Execute multiple instructions in one transaction

Compose several instructions into a single atomic transaction.

const [ix1, ix2, ix3] = await Promise.all([
  solana.hydrateAndEncodeInstruction(programA, "instructionA", optionsA),
  solana.hydrateAndEncodeInstruction(programB, "instructionB", optionsB),
  solana.hydrateAndEncodeInstruction(programC, "instructionC", optionsC),
].map(p => p.then(r => r.instructionRequest)));

const { transactionHandle } = await solana.prepareAndExecuteTransaction(
  payer,
  [ix1, ix2, ix3]  // executed atomically
);
console.log("All 3 instructions in one tx:", transactionHandle);
04 Sign with a browser wallet

Use a browser wallet (Phantom, Backpack, …) as the transaction signer.

import { Solana, walletProviders } from "solana-kiss";

// Subscribe to available wallet providers
walletProviders.subscribe(async (providers) => {
  const phantom = providers.find(p => p.name === "Phantom");
  if (!phantom) return;

  // Connect and get the active account
  const [account] = await phantom.connect();

  const solana = new Solana("mainnet");
  const { instructionRequest } = await solana.hydrateAndEncodeInstruction(
    programAddress, "myInstruction", options
  );

  // WalletAccount can be used directly as the payer/signer
  const { transactionHandle } = await solana.prepareAndExecuteTransaction(
    account,          // WalletAccount — signs via the wallet extension
    [instructionRequest]
  );
  console.log("Tx:", transactionHandle);
});
