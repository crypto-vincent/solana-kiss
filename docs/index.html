solana-kiss Cookbook

A practical guide to building on Solana with zero boilerplate —
no dependencies, full-featured, fully typed.

TypeScript Zero Dependencies Ed25519 Built-in IDL Auto-decode

Installation

npm install solana-kiss

Quick Start

The fastest path to interacting with Solana. Import the high-level Solana class and point it at a network moniker. That's it — no wallet adapters, no big SDK to configure. 

import { Solana } from "solana-kiss";

// Connect to devnet (also accepts "mainnet", "testnet", or a custom URL)
const solana = new Solana("devnet");

// Fetch and auto-decode any on-chain account
const { accountState } = await solana.getAndInferAndDecodeAccount(
  "TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA" as any
);
console.log(accountState);

Module Overview

solana-kiss ships four tightly-coupled modules. Click any card below to jump to its dedicated cookbook page.

⚡
Solana Class
High-level entry point for all on-chain operations — accounts, instructions, transactions, PDAs.
 🔑
Data
Core primitives — Pubkeys, Signers, Wallets, Transactions, encoding utilities, and cryptography.
 🌐
RPC
JSON-RPC HTTP client plus helpers for account queries, block hashes, and transaction submission.
 📋
IDL
Load, parse and use Anchor / native on-chain IDLs to auto-encode and decode instructions and accounts.

Key Features

  • Zero production dependencies — Ed25519, SHA-256, Base58 and zlib are bundled natively.
  • Full TypeScript coverage — every exported type has JSDoc, generics, and strict null-safety.
  • IDL-driven everything — accounts and instructions are automatically decoded using Anchor IDLs loaded on-chain, from a remote URL, or from a custom override.
  • Wallet Standard integration — discover browser wallets and sign transactions without a framework.
  • Composable RPC wrappers — timeout, retry, rate-limit, and concurrency-limit are standalone composable functions.
  • Versioned transactions — address lookup tables are supported out of the box.

Full Example — Send SOL

The following example shows how to generate a new keypair, build a SOL transfer using the System Program, and execute it on devnet — in under 20 lines of code. 

import { Solana, signerGenerate } from "solana-kiss";

const solana = new Solana("devnet");

// 1. Generate an ephemeral payer (in prod, load from a stored secret)
const payer = await signerGenerate();

// 2. Build the transfer instruction via IDL
const { instructionRequest } = await solana.hydrateAndEncodeInstruction(
  "11111111111111111111111111111111",   // System Program
  "transfer",
  {
    instructionAddresses: {
      fundingAccount: payer.address,
      recipientAccount: "recipient_pubkey_here" as any,
    },
    instructionPayload: { lamports: 500_000n },
  }
);

// 3. Sign, send, and wait for confirmation in one call
const { transactionHandle } = await solana.prepareAndExecuteTransaction(
  payer,
  [instructionRequest]
);

console.log("Confirmed:", transactionHandle);

TypeScript Setup

solana-kiss ships with declaration files. For best results set "moduleResolution": "bundler" (or "node16") and "strict": true in your tsconfig.json. 

{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "lib": ["ES2020", "DOM"]
  }
}
