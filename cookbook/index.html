Chapter 1 Getting Started

Install

One package. Zero peer dependencies.

$ npm install solana-kiss

Connect

Pass a moniker ("mainnet", "devnet", "testnet") or any RPC URL string directly to new Solana(). That's all the setup you need.

TypeScript
import { Solana } from "solana-kiss";

// moniker shortcut — confirmed commitment, public endpoint
const solana = new Solana("mainnet");

// or bring your own RPC URL
const solana = new Solana("https://my-rpc.example.com");

Chapter 2 The Solana Class

The high-level entry point. Wraps RPC + IDL loading so you can focus on what your program does, not on boilerplate.

Constructor

Accepts a moniker, a URL string, or a pre-built RpcHttp client. An optional idlOverrides map lets you pin specific programs to local IDL files from the start.

TypeScript
import {
  IdlProgram,
  rpcHttpFromUrl,
  Solana,
  urlRpcPublicMainnet,
} from "solana-kiss";

// 1. moniker (simplest)
const solana = new Solana("mainnet");

// 2. custom RPC client (add rate-limits, headers, …)
const rpc = rpcHttpFromUrl(urlRpcPublicMainnet, {
  commitmentLevel: "finalized",
  extraRequestHeaders: { "x-api-key": "my-key" },
});
const solana = new Solana(rpc);

// 3. pin an IDL at construction time
const localIdl: IdlProgram = loadMyIdl();
const solana = new Solana("devnet", {
  idlOverrides: new Map([[programAddress, localIdl]]),
});

Override a Program IDL

Pin a local IDL for any program address at any time. Pass undefined to remove the override and fall back to the auto-loader.

TypeScript
import localIdlJson from "./my_program.json";
import { idlProgramParse, pubkeyFromBase58, Solana } from "solana-kiss";

const solana      = new Solana("devnet");
const programAddr = pubkeyFromBase58("MyProgram1111111111111111111111111111111111");

// pin a local IDL — no on-chain fetch for this program
solana.setProgramIdlOverride(programAddr, idlProgramParse(localIdlJson));

// later: remove the override and let the auto-loader take over
solana.setProgramIdlOverride(programAddr, undefined);

Fetch & Decode an Account

Give it an address. It fetches the account, auto-loads the owning program's IDL, infers the account type, and returns fully decoded state — in one call.

TypeScript
import { pubkeyFromBase58, Solana } from "solana-kiss";

const solana  = new Solana("mainnet");
const address = pubkeyFromBase58("YourAccountAddress...");

const {
  accountState,      // decoded JSON (fully typed by IDL)
  accountIdl,        // the matched IDL account definition
  accountLamports,   // bigint
  programIdl,        // full IDL of the owning program
} = await solana.getAndInferAndDecodeAccount(address);

console.log(accountState);

Infer & Decode an Instruction

Reverse-parse a raw InstructionRequest back into human-readable named addresses and a JSON argument payload.

TypeScript
import { InstructionRequest, pubkeyFromBase58, Solana } from "solana-kiss";

const solana: Solana;
const rawInstruction: InstructionRequest; // from a decoded transaction

const {
  instructionIdl,      // matched IDL instruction definition
  instructionAddresses,// { [accountName]: Pubkey }
  instructionPayload,  // { [argName]: value }  (JSON)
} = await solana.inferAndDecodeInstruction(rawInstruction);

console.log(instructionIdl.name);    // e.g. "transfer"
console.log(instructionAddresses);    // { source: "...", dest: "..." }
console.log(instructionPayload);      // { amount: "1000000" }

Encode an Instruction

Build a ready-to-send InstructionRequest from named addresses and a JSON payload. Missing accounts (PDAs, associated token accounts) are derived automatically.

TypeScript
import { pubkeyFromBase58, Solana } from "solana-kiss";

const solana     = new Solana("mainnet");
const tokenProgram = pubkeyFromBase58("TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA");

const { instructionRequest } = await solana.hydrateAndEncodeInstruction(
  tokenProgram,
  "transfer",
  {
    instructionAddresses: {
      source: pubkeyFromBase58("SourceTokenAccount..."),
      destination: pubkeyFromBase58("DestTokenAccount..."),
      authority: signer.address,
    },
    instructionPayload: { amount: "1000000" },
  },
);

// instructionRequest is ready for prepareAndExecuteTransaction

Resolve Missing Accounts

Fill in only the accounts you know; let hydrateInstructionAddresses derive the rest from on-chain state, PDAs, or associated token account rules.

TypeScript
const { instructionAddresses } = await solana.hydrateInstructionAddresses(
  programAddress,
  "stake",
  {
    throwOnMissing: true,
    instructionAddresses: { wallet: signer.address },
    instructionPayload:   { amount: "5000000000" },
  },
);
// instructionAddresses now contains all resolved accounts

Get a Recent Block Hash

The result is cached for 15 seconds so multiple calls in the same window don't burn extra RPC requests.

TypeScript
const blockHash = await solana.getRecentBlockHash();
// BlockHash string — reused until cache expires (default 15 s)

Execute a Transaction

One call to compile, sign, send, and confirm. Block hash fetching, address lookup tables, and multi-signer flows are all handled automatically.

TypeScript
import { pubkeyFromBase58, signerFromSecret, Solana } from "solana-kiss";

const solana = new Solana("devnet");
const payer  = await signerFromSecret(secretBytes);

const {
  transactionHandle, // on-chain signature (base58)
  executionReport,   // fees, logs, error, slot
  executionFlow,     // structured call-stack trace
} = await solana.prepareAndExecuteTransaction(payer, [
  instructionRequest,
]);

if (executionReport.transactionError !== null) {
  throw new Error(`Transaction failed: ${executionReport.transactionError}`);
}
console.log("Signature:", transactionHandle);

Simulate a Transaction

Dry-run a transaction without broadcasting. Pass verifySignaturesAndBlockHash: false to simulate read-only flows where no signer is available.

TypeScript
const { executionReport } = await solana.prepareAndSimulateTransaction(
  payer,
  [instructionRequest],
);

console.log(executionReport.transactionError);  // null → would succeed
console.log(executionReport.transactionLogs);   // program log output

// read-only simulation — no signer required
const { executionReport } = await solana.prepareAndSimulateTransaction(
  payerAddress, // Pubkey only
  [instructionRequest],
  { verifySignaturesAndBlockHash: false },
);

Find Program-Owned Accounts

Returns every account owned by a program that matches a specific IDL account type. Discriminator and size filters are derived from the IDL automatically.

TypeScript
const accounts = await solana.findProgramOwnedAccounts(
  programAddress,
  "UserAccount", // IDL account type name (camelCase)
);

for (const acc of accounts) {
  console.log(acc.accountAddress, acc.accountLamports);
}

Derive a PDA via IDL

Uses the PDA definition in the program's IDL to derive the address automatically from named seed inputs — no manual seed assembly.

TypeScript
const pdaAddress = await solana.findPdaAddress(
  programAddress,
  "userProfile",       // PDA name as declared in IDL
  { wallet: signer.address },
);
console.log(pdaAddress); // base58 off-curve address

Chapter 3 data — Primitives

Low-level building blocks: keypairs, public keys, PDAs, lamports, transactions, and browser wallet integration. Import anything you need directly from "solana-kiss".

Keypairs

signerGenerate creates a fresh Ed25519 keypair using the Web Crypto API. signerFromSecret restores one from a 64-byte secret (seed + pubkey).

TypeScript
import { signerFromSecret, signerGenerate } from "solana-kiss";

// generate — private key is non-extractable (Web Crypto)
const signer = await signerGenerate();
console.log(signer.address); // Pubkey (base58)

// restore from a 64-byte keypair array
const secretBytes = new Uint8Array([/* 64 bytes */]);
const signer = await signerFromSecret(secretBytes);

// sign arbitrary bytes
const signature = await signer.sign(new Uint8Array([1, 2, 3]));

Public Keys

Pubkey is a branded base58 string — zero overhead at runtime, full type safety at compile time.

TypeScript
import {
  pubkeyFromBase58,
  pubkeyFromBytes,
  pubkeyToBytes,
  pubkeyCreateFromSeed,
} from "solana-kiss";

// parse
const pk = pubkeyFromBase58("TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA");

// from raw bytes (32 bytes)
const pk = pubkeyFromBytes(rawBytes);

// to bytes
const bytes = pubkeyToBytes(pk); // Uint8Array (32)

// derive with seed (createWithSeed convention)
const derived = pubkeyCreateFromSeed(baseAddress, "my-seed", ownerProgram);

Program Derived Addresses

Pass the program address and an ordered array of seed byte buffers. pubkeyFindPdaAddressAndBump also returns the bump seed used.

TypeScript
import {
  pubkeyFindPdaAddress,
  pubkeyFindPdaAddressAndBump,
  pubkeyFromBase58,
  pubkeyToBytes,
  utf8Encode,
} from "solana-kiss";

const program = pubkeyFromBase58("TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA");
const wallet  = pubkeyFromBase58("YourWallet...");
const mint    = pubkeyFromBase58("MintAddress...");

// derive — seeds are Uint8Arrays
const pda = pubkeyFindPdaAddress(program, [
  pubkeyToBytes(wallet),
  utf8Encode("vault"),
  pubkeyToBytes(mint),
]);

// also get the bump
const { address, bump } = pubkeyFindPdaAddressAndBump(program, [utf8Encode("vault")]);

Lamports & Fees

Lamports are bigint throughout the library. Helper functions convert to/from SOL and compute rent-exemption minimums.

TypeScript
import {
  approximateSolsForLamports,
  approximateLamportsForSols,
  lamportsRentExemptionMinimumForSpace,
  lamportsFeePerSignature,
} from "solana-kiss";

// bigint → approximate SOL (floating-point)
approximateSolsForLamports(1_500_000_000n); // 1.5

// SOL → bigint lamports
approximateLamportsForSols(1.5); // 1500000000n

// rent-exemption minimum for an account with N bytes of data
const rentMin = lamportsRentExemptionMinimumForSpace(165); // bigint

// fee per signature (5000 lamports)
const fee = lamportsFeePerSignature; // 5000n

Compile & Sign a Transaction

Low-level helpers when you want full control over the transaction lifecycle without using the Solana class.

TypeScript
import {
  rpcHttpFromUrl,
  rpcHttpGetLatestBlockHash,
  rpcHttpSendTransaction,
  rpcHttpWaitForTransaction,
  signerFromSecret,
  transactionCompileAndSign,
  urlRpcPublicDevnet,
} from "solana-kiss";

const rpc    = rpcHttpFromUrl(urlRpcPublicDevnet);
const signer = await signerFromSecret(secretBytes);

// 1. fetch recent block hash
const { blockHash } = await rpcHttpGetLatestBlockHash(rpc);

// 2. compile + sign (returns a TransactionPacket)
const packet = await transactionCompileAndSign([signer], {
  payerAddress: signer.address,
  recentBlockHash: blockHash,
  instructionsRequests: [myInstruction],
});

// 3. send
const { transactionHandle } = await rpcHttpSendTransaction(rpc, packet);

// 4. confirm
const { executionReport } = await rpcHttpWaitForTransaction(
  rpc,
  transactionHandle,
  async () => {
    await new Promise(resolve => setTimeout(resolve, 2000));
    return true;
  },
);

Browser Wallet Provider

Discover injected browser wallets (Phantom, Backpack, …) using the Wallet Standard. walletProviders is a reactive observable — subscribe once and it updates automatically.

TypeScript
import { walletProviders } from "solana-kiss";

// subscribe — fires immediately with discovered wallets
const unsubscribe = walletProviders.subscribe((providers) => {
  for (const p of providers) {
    console.log(p.name, p.icon); // "Phantom", data:image/...
  }
});

// connect the first available wallet
const [provider] = await new Promise(resolve =>
  walletProviders.subscribe((ps) => ps.length > 0 && resolve(ps)),
);
const [account] = await provider.connect();

// WalletAccount implements signMessage + signTransaction
const sig = await account.signMessage(new TextEncoder().encode("hello"));

Chapter 4 rpc — Raw RPC Calls

Thin, typed wrappers around Solana JSON-RPC methods. Each function takes an RpcHttp client as its first argument, making them trivially composable with the middleware stack.

Create an RPC Client

rpcHttpFromUrl binds a JSON-RPC function to a URL. Optionally set commitment level, custom headers, or a custom fetch implementation.

TypeScript
import {
  rpcHttpFromUrl,
  urlRpcPublicMainnet,
  urlRpcPublicDevnet,
} from "solana-kiss";

// public endpoint, confirmed commitment (default)
const rpc = rpcHttpFromUrl(urlRpcPublicDevnet);

// private endpoint with auth header + finalized commitment
const rpc = rpcHttpFromUrl(new URL("https://my-rpc.example.com"), {
  commitmentLevel: "finalized",
  extraRequestHeaders: { "Authorization": "Bearer my-token" },
});

Middleware

Wrap any RpcHttp client to add timeouts, concurrency caps, rate limiting, and automatic retry — composable in any order.

TypeScript
import {
  rpcHttpFromUrl,
  rpcHttpWithConcurrentRequestsLimit,
  rpcHttpWithRequestsPerSecondLimit,
  rpcHttpWithRetryOnError,
  rpcHttpWithTimeout,
  urlRpcPublicMainnet,
} from "solana-kiss";

let rpc = rpcHttpFromUrl(urlRpcPublicMainnet);

// add 10-second timeout per request
rpc = rpcHttpWithTimeout(rpc, 10_000);

// cap concurrent in-flight requests
rpc = rpcHttpWithConcurrentRequestsLimit(rpc, 5);

// rate-limit to 50 requests/second
rpc = rpcHttpWithRequestsPerSecondLimit(rpc, 50);

// retry on transient errors (up to 3 times)
rpc = rpcHttpWithRetryOnError(rpc, async ({ retriedCounter }) => {
  if (retriedCounter >= 3) return false;
  await new Promise(resolve => setTimeout(resolve, 500 * (retriedCounter + 1)));
  return true;
});

Fetch Account Metadata

Get owner, lamports, executable flag, and data size without pulling the full account data. Returns zeroed defaults for non-existent accounts.

TypeScript
import {
  pubkeyFromBase58,
  rpcHttpFromUrl,
  rpcHttpGetAccountMetadata,
  urlRpcPublicMainnet,
} from "solana-kiss";

const rpc     = rpcHttpFromUrl(urlRpcPublicMainnet);
const address = pubkeyFromBase58("YourAccountAddress...");

const {
  programAddress,   // Pubkey of the owning program
  accountExecutable,// boolean
  accountLamports,  // bigint
  accountSpace,     // number (bytes)
} = await rpcHttpGetAccountMetadata(rpc, address);

Send & Confirm a Transaction

rpcHttpSendTransaction broadcasts a signed packet. rpcHttpWaitForTransaction polls until confirmed, calling your retryApprover between each attempt.

TypeScript
import {
  rpcHttpSendTransaction,
  rpcHttpWaitForTransaction,
} from "solana-kiss";

const { transactionHandle } = await rpcHttpSendTransaction(rpc, packet, {
  skipPreflight: false,
});

const { transactionRequest, executionReport, executionFlow } =
  await rpcHttpWaitForTransaction(
    rpc,
    transactionHandle,
    async ({ retriedCounter, totalDurationMs }) => {
      if (totalDurationMs > 30_000) return false; // give up after 30 s
      await new Promise(resolve => setTimeout(resolve, 2000));
      return true;
    },
  );

Simulate a Transaction

Optionally request post-simulation state of up to 3 accounts. Disable signature verification for read-only probing.

TypeScript
import { rpcHttpSimulateTransaction } from "solana-kiss";

const { executionReport, simulatedAccountsByAddress } =
  await rpcHttpSimulateTransaction(rpc, signedPacket, {
    verifySignaturesAndBlockHash: false,
    simulatedAccountsAddresses: new Set([vaultAddress]),
  });

console.log(executionReport.unitsConsumed); // compute units used

Fetch a Transaction

Returns undefined if the transaction is not yet confirmed. Returns the full request, execution report, and optional call-stack flow once found.

TypeScript
import {
  rpcHttpGetTransaction,
  transactionHandleFromBase58,
} from "solana-kiss";

const handle = transactionHandleFromBase58("5fdr7...base58sig");
const result = await rpcHttpGetTransaction(rpc, handle);

if (result === undefined) {
  console.log("not confirmed yet");
} else {
  const { transactionRequest, executionReport, executionFlow } = result;
  console.log(executionReport.chargedFeesLamports); // bigint
}

Fetch Account Transaction History

Paginates through an account's history, newest-first, up to maxResultLength. Supports resume and rewind cursors.

TypeScript
import {
  pubkeyFromBase58,
  rpcHttpFindAccountTransactions,
} from "solana-kiss";

const { newToOldTransactionsHandles } =
  await rpcHttpFindAccountTransactions(
    rpc,
    pubkeyFromBase58("YourAccount..."),
    50, // max results
  );

for (const handle of newToOldTransactionsHandles) {
  console.log(handle); // TransactionHandle (base58)
}

Find Program-Owned Accounts

Low-level getProgramAccounts call with optional data-size and memory-pattern filters (max 4 filters, Solana RPC limit).

TypeScript
import {
  rpcHttpFindProgramOwnedAccounts,
  utf8Encode,
} from "solana-kiss";

const accounts = await rpcHttpFindProgramOwnedAccounts(rpc, programAddress, {
  dataSpace: 165,                         // exact byte size
  dataBlobs: [
    { offset: 0, bytes: discriminator },  // memcmp filter at byte 0
  ],
});

for (const acc of accounts) {
  console.log(acc.accountAddress, acc.accountSpace);
}

Chapter 5 idl — Parsing & Codec

Parse Anchor IDL JSON, encode and decode instruction arguments and account data. These are the building blocks the Solana class uses internally.

Load an IDL

Combine multiple loaders in a fallback sequence. Wrap any loader with memoization to avoid re-fetching the same program twice.

TypeScript
import {
  idlLoaderFromLoaderSequence,
  idlLoaderFromUrl,
  idlLoaderMemoized,
  idlProgramParse,
  pubkeyFromBase58,
} from "solana-kiss";

// loader from a URL template
const urlLoader = idlLoaderFromUrl(
  (programAddress) => new URL(`https://my-cdn.example.com/idl/${programAddress}.json`),
);

// fallback sequence: CDN first, then on-chain
const loader = idlLoaderMemoized(
  idlLoaderFromLoaderSequence([urlLoader, onchainLoader]),
);

// load manually
const programIdl = await loader(pubkeyFromBase58("MyProgram..."));

// or parse a local JSON object
const programIdl = idlProgramParse(localIdlJson);

Encode Instruction Arguments

Turn a JSON payload and named account addresses into the raw bytes expected by the on-chain program.

TypeScript
import {
  idlInstructionAccountsEncode,
  idlInstructionArgsEncode,
  idlProgramParse,
} from "solana-kiss";

const programIdl     = idlProgramParse(idlJson);
const instructionIdl = programIdl.instructions.get("transfer")!;

// encode accounts → ordered InstructionInput array
const { instructionInputs } = idlInstructionAccountsEncode(instructionIdl, {
  source: sourceAddress,
  destination: destAddress,
  authority: signer.address,
});

// encode args → Uint8Array (discriminator + serialised args)
const { instructionData } = idlInstructionArgsEncode(instructionIdl, {
  amount: "1000000",
});

Decode Instruction Arguments

Parse a raw InstructionRequest back into named account addresses and a JSON payload using the IDL schema.

TypeScript
import {
  idlInstructionAccountsDecode,
  idlInstructionArgsDecode,
} from "solana-kiss";

const { instructionAddresses } = idlInstructionAccountsDecode(
  instructionIdl,
  rawInstruction.instructionInputs,
);
// { source: Pubkey, destination: Pubkey, authority: Pubkey }

const { instructionPayload } = idlInstructionArgsDecode(
  instructionIdl,
  rawInstruction.instructionData,
);
// { amount: "1000000" }

Encode & Decode Account Data

Convert raw account bytes to a typed JSON object (decode) or serialise account state back to bytes (encode) using an IDL account definition.

TypeScript
import {
  idlAccountDecode,
  idlAccountEncode,
  idlProgramParse,
} from "solana-kiss";

const programIdl = idlProgramParse(idlJson);
const accountIdl = programIdl.accounts.get("UserAccount")!;

// decode bytes → JSON state
const { accountState } = idlAccountDecode(accountIdl, rawAccountBytes);
console.log(accountState); // { owner: "...", balance: "1000", ... }

// encode JSON state → bytes
const { accountData } = idlAccountEncode(accountIdl, {
  owner: ownerAddress,
  balance: "1000",
});
